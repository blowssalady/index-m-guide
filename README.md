# index-m-guide

## 项目简介

index-m-guide 是一个用于归档和发布多个独立 HTML 页面的仓库。这里存放的页面均为静态内容，涵盖信息展示、工具说明、参考文档等多种用途。项目本身不绑定特定域名或网站，仅作为 HTML 文件的结构化存储与访问入口。

## 目录结构

```
index-m-guide/
├── pages/          # 存放各个独立 HTML 页面
├── assets/         # 公共资源文件（CSS、JS、图片等）
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹对应一个独立页面，文件夹内包含该页面的 HTML 及相关资源。
- `assets/` 目录用于存放多个页面共用的样式、脚本或图片，以减少重复。

## 页面归档说明

- 所有页面均为纯前端静态 HTML，不依赖后端服务。
- 每个页面独立完整，可直接在浏览器中打开运行。
- 页面内容遵循通用信息规范，不针对特定域名或实体。
- 归档时请保持文件结构清晰，并在 `pages/` 下创建独立子目录。

## 维护说明

- 欢迎提交 Pull Request 添加新的 HTML 页面，或对现有页面进行改进。
- 提交前请确保页面无外部依赖或已将所有依赖内联或放入 `assets/` 目录。
- 请勿在页面中包含恶意代码、跟踪脚本或违反平台政策的内容。
- 本仓库仅作为归档与展示用途，不提供在线托管以外的服务承诺。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下任意页面的 HTML 文件。
3. 也可通过 GitHub Pages 等服务部署后在线访问（需自行配置）。

## 贡献

如有新页面希望归档，请按以下步骤操作：

1. Fork 本仓库。
2. 在 `pages/` 下创建以页面功能命名的文件夹。
3. 将页面文件（HTML、CSS、JS 等）放入该文件夹。
4. 提交 Pull Request，并简要说明页面用途。

## 许可

本仓库内容采用通用开放许可，具体页面可能附带各自标注的授权信息。如无特殊说明，默认允许合理使用和转载，但请保留来源。
